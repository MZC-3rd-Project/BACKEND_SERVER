# GitHub - Jira  연동


## 개요

* **단방향 동기화**: GitHub에서 발생한 이벤트(생성, 수정, 닫기, 재오픈)만 Jira에 반영됨
* **Jira 직접 수정 금지**: 현재 워크플로우는 Jira 보드에서 이슈를 직접 생성하거나 상태를 변경해도 GitHub에 반영되지 않음,,,


---

## Issue 계층 구조 및 작성법

프로젝트 관리를 위해 **Epic - Story - Task**의 3단계 계층 구조를 한 묶음으로 작성하는 것을 권장

### 1. 업무 유형(Issue Types) 정의
| 유형 | 설명 | 제목 규칙 |
| :--- | :--- | :--- |
| **Epic** | 큰 단위의 기능 목표 (예: 사용자 인증 시스템) | `[EPIC] 제목` |
| **Story** | 사용자가 얻는 가치 중심의 요구사항 | `[STORY] 제목` |
| **Task** | 실제 개발 및 테스트 단위 작업 (Java, Spring Boot 개발 등) | `[TASK] 제목` |
| **Bugfix** | 버그 발견 및 수정, 결함 해결 | `[BUGFIX] 제목` |

### 2. Issue 연결 
이슈 생성 시 템플릿의 **`Related Story/Epic`** 필드를 활용하세요.
* **작성법**: `- Related #issue번호` 형식으로 상위 이슈 번호를 입력합니다. (예: `#12`)
* **결과**: Jira에서 해당 이슈 간에 'Relates' 링크가 자동으로 생성되어 관계 파악 가능

##  Jira issue 자동 동기화

### 1. Issue 상태 전이
자동 동기화: `To Do`(생성/재오픈), `Done`(닫기/머지) 상태는 GitHub 활동에 따라 자동으로 전환됨
* **Issue 생성**: Jira 이슈가 자동으로 생성되며 `To Do` 상태가 됨
* **Issue 닫기(Close)**: Jira 이슈가 자동으로 `Done`으로 전환
* **Issue 재오픈(Reopen)**: Jira 이슈가 다시 `To Do` 상태로 복구


### 2. 잔행중 상태 관리 및 모니터링
Jira에서의 `In Progress(진행 중)` 상태 전환은 자동화되어 있지 않음

=> Jira에서는 팀원들이 현재 어떤 작업이 완료되었고, 어떤 작업이 남았는지 **전체적인 진행률을 확인하는 용도**로 사용

 - **사용법** : Jira 보드에서 본인의 작업 상태를 직접 `In Progress`로 옮겨 팀원들에게 공유 -> 최종 완료 처리는 GitHub Issue를 Close 


### 3. PR merge를 통한 자동 완료 (PR Sync)
PR을 통해 작업을 완료할 때는 아래의 **Keywords**를 사용

* **사용법**: PR 본문에 `- Closes #이슈번호` 또는 `- Fixes #이슈번호`를 입력

PR이 `develop` 또는 `main` 브랜치에 merge 되면 :

1. 연결된 GitHub 이슈가 자동으로 **Closed** 처리
2. 연동된 Jira 이슈가 자동으로 **Done** 상태로 전환
